# Proposal: Fix Tauri Command Registration

## Intent

8 Tauri commands are fully implemented in Rust but never registered. `generate_handler![]` only includes `greet`, so every frontend invoke silently fails with "command not found". **"Abrir proyecto" does nothing** because `open_folder_dialog` is unreachable.

## Scope

### In Scope
- Add `mod commands;` to `src-tauri/src/lib.rs`
- Register all 8 commands in `generate_handler![]`: `read_file`, `write_file`, `list_dir`, `create_dir`, `delete_entry`, `open_folder_dialog`, `validate_project`, `exec_shell`
- Verify Rust compilation (`cargo check`) and TypeScript typecheck pass

### Out of Scope
- Frontend changes
- New commands
- Web platform support
- Testing framework for Tauri commands (separate cycle)

## Capabilities

### New Capabilities
None — all capabilities already have specs.

### Modified Capabilities
None — existing specs (`file-system`, `desktop-shell`, `terminal`) already describe correct behavior. This is a pure registration fix.

## Approach

Two-line fix in `src-tauri/src/lib.rs`:

1. Add `mod commands;` after the `use tauri::Manager;` import so the command modules are compiled into the crate.
2. Expand `generate_handler![greet]` to include all 8 commands: `commands::fs::read_file`, `commands::fs::write_file`, `commands::fs::list_dir`, `commands::fs::create_dir`, `commands::fs::delete_entry`, `commands::project::open_folder_dialog`, `commands::project::validate_project`, `commands::shell::exec_shell`.

The commands are already `pub` and `#[tauri::command]` annotated — they just need registration.

## Affected Areas

| Area | Impact | Description |
|------|--------|-------------|
| `src-tauri/src/lib.rs` | Modified | Add `mod commands;` + expand `generate_handler![]` |

## Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| `open_folder_dialog` fails at runtime (async + `AppHandle` injection) | Low | Tauri injects `AppHandle` for `async` commands automatically; `cargo check` validates signature |
| `exec_shell` panics on malformed input | Low | Command already has error handling; registration itself adds no new risk |
| Compile error from unused import | Low | Only `commands::fs` defines serde types (`FileEntry`, `ShellOutput`) — no unused import generated by registration |

## Rollback Plan

Revert `lib.rs` to original: remove `mod commands;` and restore `generate_handler![greet]`. Single file, clean revert.

## Dependencies

None. No external crates, no config changes, no migration.

## Success Criteria

- [ ] `cargo check` passes in `src-tauri/`
- [ ] `npm run typecheck` passes
- [ ] `generate_handler![]` lists all 8 commands
- [ ] `mod commands;` is present in `lib.rs`
